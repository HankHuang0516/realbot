如何免費使用 OpenClaw — E-Claw App 設定教學

2026 年 2 月 ｜ 圖文教學

OpenClaw 是什麼？

OpenClaw 是一個 AI Agent，可以理解為一個能夠自主執行任務的 AI 助手，而不只是單純的語言模型。透過 E-Claw 這款 App，你可以免費使用 OpenClaw 提供的功能。

具體來說，E-Claw 的「官方 Bot 租用」功能提供了一個免費方案，搭載 MiniMax 2.1 模型，每天有 15 則訊息的額度。以下是完整的設定流程。

設定步驟

1. 下載 E-Claw App

到 Google Play Store 搜尋「e-claw」，安裝 E-Claw 電子蝦桌布 (OpenClaw)。

下載連結放在文末。

2. 找到「官方 Bot 租用」

打開 App 後在主畫面往下滑，找到「官方 Bot 租用」按鈕並點進去。

3. 選擇免費版

進入租用頁面後會看到免費版和月租版兩個選項，選「免費版」然後點「使用免費版」。

免費版使用 MiniMax 2.1 模型，與所有免費用戶共享記憶，每天 15 則訊息。

4. 等待伺服器連接

點擊後 App 會開始連接伺服器，畫面會顯示「正在與機器人交握中...」，等一下就好。

5. 完成連接流程

連接過程中畫面會跑一些 Webhook 測試訊息，建議等流程跑完再離開頁面。

連接過程中不要關閉 App 或切到其他頁面，不然可能會連接失敗。

6. 看到龍蝦圖示就完成了

畫面出現龍蝦圖示並顯示「Connected!」或「成功啦」，代表連接成功，可以回主頁開始用了。

開始使用

回到 E-Claw 主頁，點「聊天」就能跟 OpenClaw 對話了。

幾點補充

  • 免費版每天 15 則訊息，用完隔天重置。
  • 記憶是所有免費用戶共用的，別丟敏感資料進去。
  • MiniMax 2.1 拿來日常對話、問問題夠用了。
  • 如果覺得不夠用，可以考慮月租版，有無限訊息和獨立記憶。

系統架構

Android App ↔ Backend ↔ OpenClaw Platform

架構總覽

E-Claw 由三個主要部分組成：Android App、後端服務（Railway）、以及 OpenClaw AI 平台（Zeabur）。

┌─────────────────────┐ ┌──────────────────────┐ │ Android App │ │ Web Portal │ │ (Kotlin) │ │ (HTML/JS) │ │ │ │ │ │ Live Wallpaper │ │ Entity Management │ │ Chat UI │◄────► Bot Config │ │ Push Receiver │ │ Telemetry Viewer │ └─────────┬───────────┘ └────────┬─────────────┘ │ │ │ HTTPS / REST API │ ▼ ▼ ┌─────────────────────────────────────────────────┐ │ Backend (Railway) │ │ Node.js + Express │ │ │ │ /api/bind /api/broadcast /api/transform │ │ /api/chat /api/logs /api/telemetry │ │ │ │ PostgreSQL (persistent store) │ └────────────────┬───────────────────┬────────────────┘ │ Webhook mode │ Channel mode │ push + exec+curl │ /api/channel/* ▼ ▼ ┌──────────────────────────────────────┐ │ OpenClaw Platform │ │ (Zeabur / Railway / self-host) │ │ │ │ AI Bot Instances (Webhook binding) │ │ Channel Plugin (Channel binding) │ │ (free: 4 slots / paid: 8 slots) │ └──────────────────────────────────────┘

核心設計

  • 每個裝置擁有 無上限的動態 entity slots — 綁定 Bot 時自動新增空位，每個 slot 可獨立綁定不同 Bot
  • Bots 接入方式：Webhook mode（push + exec+curl）或 Channel plugin mode（/api/channel/*），可混用
  • Railway 在 push 到 main 時自動部署（監控 backend/ 資料夾）

快速開始

本地開發、部署、Android App 安裝

先決條件

  • Android 8.0+ 裝置

# Clone the repo git clone https://github.com/HankHuang0516/realbot.git cd realbot/backend # Install dependencies npm install # Configure environment cp .env.example .env # then fill in DATABASE_URL, etc. # Start dev server npm run dev # → Server running on http://localhost:3000

# Connect repo to Railway (auto-deploys from backend/ on push to main) # Set environment variables in Railway dashboard: # DATABASE_URL PORT NODE_ENV git push origin main

Android App 安裝

  1. 從 GitHub Releases 下載最新的 .aab / .apk
  2. 設定為 Live Wallpaper → Long Settings → 輸入你的 deviceId
  3. 打開 Web Portal 綁定 AI 實體

Dashboard 同步機制

Mission Control Dashboard 使用版本號（version）做樂觀鎖定。每次操作都會自動遞增版本號，Bot 可透過定期輪詢 GET /api/mission/dashboard 比對版本號來偵測使用者的靜默修改。

建議 Bot 每 15 分鐘輪詢一次 Dashboard，比對 version 欄位，若版本變更則重新讀取所有項目。這能確保 Bot 即時回應使用者在 Web 面板上的操作。

環境變數 (Env Variables)

任務中心 > 環境變數 User 授權 / Bot 讀取

讓 Bot 安全存取敏感資訊（API Key、Token 等）。變數以 AES-256-GCM 加密儲存於伺服器，Bot 每次讀取都需要裝置擁有者即時授權（JIT — Just-in-Time Approval）。

功能明細

功能說明
加密儲存變數值以 AES-256-GCM 加密，伺服器僅存密文，Key 名稱可見但值不可見
JIT 即時授權Bot 請求讀取時，App 彈出授權對話框，使用者確認後才解密回傳，授權快取 5 分鐘
鎖定保護開啟鎖定後，Bot 的所有讀取請求立即拒絕，不顯示授權彈窗
離線保護裝置未連線（Socket.IO 斷線）時，Bot 請求直接回傳 403，不會等待
Web 管理可在 Portal → Env Variables 頁面新增、編輯、刪除變數
App 管理在 App → Mission Control → Env Variables 同樣可管理，並接收授權通知

授權流程

Bot 呼叫 GET /api/device-vars ↓ 伺服器確認變數存在 ↓ 透過 Socket.IO 推送授權請求到 App ↓ App 顯示 Material Dialog（變數名稱清單） 使用者點「允許」 ↓ 伺服器解密 → 回傳變數值給 Bot ↓ 授權快取 5 分鐘（同 Bot 再次請求免彈窗）

操作API

遠端控制 (Remote Control)

任務中心 > 遠端控制 User 啟用 / Bot 執行

讓 Bot 自主操控你的手機 UI。Bot 透過 Accessibility Tree（無障礙語意樹） 讀取畫面結構，再發送點擊、輸入、滑動等指令。整個過程不截圖，Bot 只看到文字元素，不會擷取敏感視覺內容。

此功能預設關閉，需在 App Settings → Remote Control 手動開啟並授予無障礙服務權限。

功能明細

功能說明
畫面擷取Bot 請求目前畫面的語意樹（最多 300 個元素），包含文字、類型、座標、可互動屬性
點擊 (tap)按 Node ID 或 XY 座標點擊元素
輸入 (type)對指定輸入欄位輸入文字
提交 (ime_action)對輸入欄位送出鍵盤 Enter / 確認，適用於沒有實體送出按鈕的情境
滑動 (scroll)對可滾動元素向上或向下滑動
系統按鍵返回鍵（back）、主畫面鍵（home）
多輪自主操作Bot 可反覆截圖 → 判斷 → 執行指令，完成複雜的多步驟任務

自帶 OpenClaw Bot — Channel 綁定教學

讓你自己的 AI（例如 Claude）住進 E-Claw 角色裡

Channel 是什麼？

E-Claw 預設的「官方 Bot」是平台提供的共用 AI。如果你有自己的 OpenClaw 伺服器（自架或 Zeabur 部署），可以透過 Channel 綁定，把你的 AI 直接接進 E-Claw 的角色裡。

綁定後，你在 App 或 Portal 裡傳給那個角色的訊息，會直接推送到你的 OpenClaw AI；AI 回覆後，角色的狀態會即時更新。

Channel 綁定是選用功能——沒有自己的 OpenClaw 的話，繼續使用官方 Bot 即可。

前置條件

  • ✅ 已有 E-Claw 帳號並登入 Portal
  • ✅ 已安裝 OpenClaw（Zeabur / Railway / 本機均可）
  • ✅ 在 OpenClaw 安裝了 openclaw-channel-eclaw plugin
  • ✅ 你的 OpenClaw 伺服器可被外網存取（或設定了 ECLAW_WEBHOOK_URL）

步驟教學

第 1 步：在 Portal 產生 API Key

前往 設定 → Channel API，點「新增 API Key」。系統會產生一組 Key + Secret，請立刻複製妥當——Secret 只顯示一次。

一個 API Key 可以同時綁定多個 entity slots（讓同一台 OpenClaw 管理多個角色）。如果你有多台獨立的 OpenClaw 伺服器，才需要分別產生不同的 Key。

第 2 步：在 OpenClaw 安裝並設定 E-Claw Channel

在你的 OpenClaw 設定（openclaw.config.yaml）加入以下 channel 設定：

channels: - type: eclaw config: channel_api_key: "eck_你的Key貼這裡" channel_api_secret: "ecs_你的Secret貼這裡" entity_id: 0 # 要綁定哪個角色（0–7） entity_name: "我的AI" # 角色顯示名稱（可選） eclaw_webhook_url: "https://你的OpenClaw網址"
ECLAW_WEBHOOK_URL 必須是外網可連到的位址，E-Claw 伺服器需要用它把用戶訊息推送過去。

第 3 步：重啟 OpenClaw

儲存設定後重啟 OpenClaw。Plugin 啟動時會自動：

  1. 向 E-Claw 伺服器登記 Callback URL
  2. 用 API Key 綁定你指定的角色（entity_id）

成功的話，OpenClaw 的 log 會出現類似 [eclaw] Bound entity 0 as "我的AI" 的訊息。

第 4 步：確認角色出現在 App / Portal

回到 E-Claw App 主畫面，或重新整理 Portal 儀表板。你指定的角色應該會顯示為已綁定，名稱也會更新為你設定的 entity_name。

角色旁邊如果有 [C] 標記，代表這是 Channel 綁定（Channel binding type）。

第 5 步：傳第一則訊息

在 App 裡點進那個角色的聊天畫面，輸入任何訊息。流程如下：

  1. E-Claw 把你的訊息推送到 OpenClaw（你設定的 ECLAW_WEBHOOK_URL）
  2. 你的 AI 處理後，呼叫 E-Claw 的 POST /api/channel/message 回覆
  3. 角色狀態更新，App 顯示 AI 的回覆

如果 AI 有回應，代表 Channel 雙向通訊正常運作。

常見問題

Q: 角色綁定後沒出現在 App？

先確認 OpenClaw log 有出現 Bound entity 成功訊息。如果沒有，通常是 API Key/Secret 貼錯、或 entity_id 超出範圍（免費方案 0–3，付費方案 0–7）。

Q: 傳訊息後 AI 沒有回應？

最常見原因是 ECLAW_WEBHOOK_URL 外網連不到。確認你的 OpenClaw 伺服器有公開的 HTTPS 端點，或透過 ngrok 等工具暴露出來。

Q: 可以同時綁定多個角色給不同的 AI 嗎？

可以。為每個 OpenClaw session 分別產生一組 API Key，各自設定不同的 entity_id，即可讓角色 0 對應 AI-A、角色 1 對應 AI-B，彼此獨立。

Q: API Key 外洩怎麼辦？

前往設定頁刪除該 Key，再產生新的一組即可。舊 Key 刪除後立即失效，對應的角色會自動解綁。

技術補充（進階）

Channel API 端點一覽（開發者用）：

# 登記 Callback URL POST /api/channel/register { channel_api_key, channel_api_secret, callback_url, callback_token } # 綁定角色 POST /api/channel/bind { channel_api_key, channel_api_secret, entityId, name } # Bot 回覆用戶（更新角色狀態） POST /api/channel/message { channel_api_key, deviceId, entityId, botSecret, message, state } # 取消登記 DELETE /api/channel/register { channel_api_key, channel_api_secret }

Frequently Asked Questions
Everything you need to know about E-Claw and the OpenClaw bot platform.
General
What is E-Claw?▼
E-Claw is a live wallpaper companion platform for Android. You can bind AI bots to your device — they update your home screen in real time with messages, mood, and animations. Manage everything through the Android app or the web portal.
What is OpenClaw?▼
OpenClaw is the bot development platform behind E-Claw. Bot developers build and host their bots on OpenClaw (powered by Zeabur), which then communicates with E-Claw devices via webhook push and exec+curl.
Is E-Claw free?▼
Yes! E-Claw offers a free tier with shared bots (up to 15 messages/day per free bot). Your own bots (self-hosted) have no message limit and are always free.
Setup & Binding
How do I bind a bot to my device?▼
Go to the Dashboard (in the app or web portal), select an entity slot, and click "Generate Code". Copy the binding command and paste it to the bot you want to bind. The bot will confirm the binding automatically.
How many bots can I bind?▼
Each device supports up to 8 entity slots (0–7). Each slot can be bound to a different bot independently with its own state, memory, and chat history.
Do I need the Android app, or can I use the web portal only?▼
Both work independently. The Android app provides the live wallpaper experience. The web portal lets you manage entities, chat, upload files, and configure missions — all from your browser. You can use either or both.
Features
What is Broadcast?▼
Broadcast lets you send a single message to all your bound entities at once. Great for announcements or updates that every bot should hear. You can track which entities received the broadcast in the delivery report.
What is Speak-to?▼
Speak-to is a direct conversation mode where you send a message to a specific entity and get a reply. It's like a one-on-one chat with your bot, with the conversation stored in chat history.
What is Mission Control?▼
Mission Control is a built-in task management system. You can create tasks (TODO/Mission/Done), assign them to entities, manage notes, configure skills and rules, and define your bot's personality ("soul"). Bots get notified when their tasks change.
Bot Development
How do I create my own bot?▼
Bots are built on the OpenClaw platform (hosted on Zeabur). Check the User Guide tab above for tutorials, API docs, and MCP skills documentation to get started.
Can I use my bot on Telegram too?▼
Yes! OpenClaw bots support multiple channels. You can add a Telegram channel adapter to your bot. See the Compare tab above for differences between E-Claw and Telegram.
Still have questions? Check the User Guide for detailed documentation.

E-Claw vs Telegram
E-Claw offers a full AI Agent collaboration experience — A2A communication, live visualization, push broadcasts, tasks, and more. Telegram only has chat.
🦞
E-Claw
Native live wallpaper channel
✈️
Telegram
Chat-based messaging channel
SETUP
📱 Android App or Web Portal
Use either the Android app or the web portal — each works independently. Register, generate binding code, paste to bot.
💬 Search Bot on Telegram
Open Telegram, search for the bot, press /start — ready to chat immediately.
INTERACTION
🔄 Push + Speak-to + Broadcast
Bot pushes status updates to wallpaper. Speak-to for direct conversation. Broadcast to reach all entities.
⌨️ Standard Chat
Send messages, receive replies. Classic chatbot experience with text, images, and inline buttons.
LIVE WALLPAPER
🎨 Full Integration
Bot controls wallpaper state (mood, message, animation). Your home screen becomes alive.
Native Support
🖼️ No Wallpaper Control
Telegram bots cannot update your phone's live wallpaper. Chat-only experience.
Unavailable
PUSH NOTIFICATIONS
🔔 Proactive Push
Bot pushes updates autonomously via API. Wallpaper changes in real-time without user action.
Real-time
📩 Message Notification
Standard Telegram notifications when bot sends a message. User needs to open app to read.
Passive
MULTI-ENTITY
👥 Up to 8 Entities
Bind different bots to different entity slots (0-7). Each has independent state and memory.
Multi-Entity
🤖 1 Bot = 1 Chat
Each Telegram bot is a separate chat. No entity slot system, but you can add multiple bots.
Manual
MEMORY & CONTEXT
🧠 Server-side Memory
Chat history stored on E-Claw server. Shared or private memory depending on plan (free/personal).
💾 Bot-managed Memory
Memory depends on bot implementation. Can use OpenClaw's built-in memory or external storage.
MESSAGE FORMAT
📐 Text + State Fields
Messages include text, mood, animation state. Structured JSON via API for wallpaper control.
🎞️ Rich Media
Text, photos, videos, files, stickers, inline keyboards, location, and more Telegram features.
PLATFORM
📱 Android + Web Portal
Live wallpaper on Android. Web portal (works on iPhone too!) for chat, mission control, files — all features except wallpaper.
🌐 All Platforms
Telegram is available on iOS, Android, desktop, and web. Chat from anywhere.
COST
💰 Free + Paid Options
Free bot (shared, 15 msg/day). Personal bot NT$288/mo (dedicated, unlimited). Self-host: free.
💸 Bot API Free
Telegram Bot API is free. You pay for hosting and LLM API usage. Full control over costs.
BOT DEVELOPMENT
🛠️ OpenClaw + Webhook
Build on OpenClaw platform (Zeabur). Use exec+curl push mode. MCP skills documentation provided.
🔌 OpenClaw + Telegram Adapter
Build on OpenClaw, add Telegram channel adapter. Uses Telegram Bot API for message routing.
MISSION CONTROL
📋 Integrated Task Hub
Built-in task board (TODO/Mission/Done), notes, skills, rules, and soul/personality system. Assign tasks to entities, push notifications on changes, version-controlled collaboration.
Built-in
🔗 No Built-in Task System
Telegram has no native task management. Need third-party bots or external tools (Trello, Notion) for project tracking. No entity-aware assignment.
Unavailable
CHAT ATTACHMENTS
📎 Photos, Voice & Files + File Manager
Upload photos (Flickr cloud), record voice messages, send files (up to 100 MB). Dedicated File Manager page with entity/type filtering, broadcast attachments, and delivery tracking.
Organized
🖼️ Rich Media in Chat
Photos, videos, files, stickers, voice and video messages — all inline in chat. Files stored on Telegram servers. No separate file manager or entity-level filtering.
Chat Only
