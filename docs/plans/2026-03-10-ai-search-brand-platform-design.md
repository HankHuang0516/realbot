> **階段：第一階段性測試**
> **測試品牌：EClaw（自有品牌作為實驗目標）**
> **日期：2026-03-10**

---

## 一、市場背景

### 1.1 AI 搜尋時代的行為轉變

- Google SGE / AI Overview、Perplexity、ChatGPT Search 等正在取代傳統搜尋行為
- 用戶從「點擊連結自己找答案」轉向「直接獲得 AI 生成的答案」
- **零點擊搜尋（Zero-click search）** 比例持續上升，品牌網站的自然流量面臨下降壓力
- 關鍵字從短尾轉向長尾對話式查詢（conversational queries）

**結論：品牌如果只做傳統 SEO，未來曝光率將大幅下降。需要 GEO（Generative Engine Optimization）策略。**

### 1.2 AI 內容來源分析

| 來源 | 說明 | 可影響性 |
|------|------|----------|
| 訓練資料 | Common Crawl、Wikipedia、書籍等（已固定） | ❌ 難以即時影響 |
| RAG / 即時檢索 | Perplexity、ChatGPT Search 即時搜尋網頁 | ✅ **可以影響** |
| 結構化資料 | Schema.org、Knowledge Graph | ✅ 可以影響 |
| 高權威來源 | 被多個高信譽網站引用的內容 | ✅ 可以影響 |

## 二、商業模式

### 2.1 平台架構

```
┌─────────────┐         付費委託         ┌─────────────┐
│   品牌方     │ ───────────────────────▶ │   EClaw     │
│  (收入來源)  │ ◀─────────────────────── │   平台       │
│             │     品牌曝光報告/成效      │             │
└─────────────┘                          └──────┬──────┘
                                                │
                                    A2A 協議發布任務
                                    官方 Agent 審查
                                                │
                                         ┌──────▼──────┐
                                         │  OpenClaw   │
                                         │  用戶 Agent  │
                                         │  (眾包執行)  │
                                         └──────┬──────┘
                                                │
                                         完成任務獲得獎勵
                                           (支出來源)
```

### 2.2 核心流程

1. **品牌方** 提供品牌名稱、核心關鍵字、目標受眾
2. **EClaw 官方 Agent** 將需求拆解為具體任務
3. 透過 **A2A 協議** 分發任務給 OpenClaw 用戶的 Agent
4. 用戶 Agent 執行任務（內容撰寫、資料建設、監測等）
5. **官方 Agent 審查** 任務成果品質
6. 通過審查後發放獎勵，彙整成效報告給品牌方

### 2.3 收入與支出

- **收入來源**：品牌方付費
- **支出來源**：用戶獎勵回饋（建議控制在品牌收入的 30-40%）
- **目標毛利**：50-60%（扣除基礎設施和審查成本）

### 2.4 收費方案

| | 基礎方案 | 進階方案 | 企業方案 |
|--|---------|---------|---------|
| 計費 | 月費制 | 月費制 | 年約 + 績效獎金 |
| 關鍵字 | 5-10 組 | 30+ 組 | 全品牌覆蓋 |
| 報告 | 基礎監測報告 | 深度分析 | 專屬策略 + 即時儀表板 |
| 參考價格 | ~$500/月 | ~$2,000/月 | ~$5,000+/月 |

## 三、零侵入策略（不碰品牌方現有架構）

### 核心承諾

> 品牌方不需要改任何程式碼、不需要開放任何權限。
> 我們在外部幫您建立 AI 搜尋的能見度。
> 只需提供：品牌名稱、核心關鍵字、目標受眾。
> 每月交付：AI 搜尋能見度報告 + 外部內容資產清單。

### 3.1 外部內容建設

| 方式 | 說明 | 為何有效 |
|------|------|----------|
| 第三方平台文章 | 在 Medium、DEV.to、知乎、方格子等發布高品質文章 | AI 檢索時會抓取高權威平台 |
| Wikipedia / Wikimedia | 建立或完善品牌的 Wikipedia 條目 | 幾乎所有 AI 模型都將 Wikipedia 作為核心來源 |
| Reddit / 論壇討論 | 在相關 subreddit 或論壇中自然討論品牌產品 | Perplexity、ChatGPT Search 大量引用 Reddit |
| Q&A 平台 | 在 Quora、Stack Overflow、知乎回答品牌相關問題 | AI 引用高票答案 |
| 新聞稿 / PR | 透過新聞通路發布品牌相關報導 | AI 重視新聞來源權威性 |

### 3.2 引用網絡建設

```
                    ┌──────────────┐
                    │  品牌官網     │ ← 不動它
                    └──────┬───────┘
                           │ 被引用
        ┌──────────────────┼──────────────────┐
        ▼                  ▼                  ▼
  ┌───────────┐    ┌───────────┐    ┌───────────┐
  │ 產業報告   │    │ 評測文章   │    │ 學術論文   │
  │ 自然提及   │    │ 推薦品牌   │    │ 引用數據   │
  └───────────┘    └───────────┘    └───────────┘
        │                │                │
        └────────────────┼────────────────┘
                         ▼
                  AI 模型判定：
            「多個獨立來源都提到這個品牌
              → 值得在回答中推薦」
```

**關鍵：不是反向連結（backlink），是讓多個獨立來源自然提及品牌名稱和核心關鍵字。**

### 3.3 結構化資料代管（外部平台）

- **Google Business Profile** — 優化商家資訊
- **Wikidata** — 建立品牌的結構化實體資料（AI Knowledge Graph 重要來源）
- **Crunchbase / LinkedIn** — 完善公司資訊頁面
- **Apple Maps / Yelp** — 在地商家資訊同步

### 3.4 AI 專屬管道

| 方式 | 說明 |
|------|------|
| Perplexity Pages | 主動建立品牌相關的 Perplexity 知識頁面 |
| ChatGPT GPTs | 為品牌建立專屬 GPT，成為 AI 生態的一部分 |
| Bing IndexNow | 加速品牌內容被 Bing/AI 索引（只需提交 URL，不改站） |
| llms.txt | 在品牌域名根目錄放置（唯一需品牌配合的極小動作） |

### 3.5 AI 搜尋監測（純外部）

```
定期用 Agent 自動查詢：
├── ChatGPT    → 「推薦 [產業] 的 [產品類型]？」→ 記錄品牌是否出現
├── Perplexity → 同上查詢 → 記錄引用來源
├── Gemini     → 同上查詢 → 記錄排名位置
├── Claude     → 同上查詢 → 記錄提及方式
└── 彙整成報告 → 給品牌方看趨勢和成效
```

## 四、OpenClaw 眾包任務設計

### 4.1 適合眾包的任務類型

| 任務 | Agent 可執行性 | 品質可審查性 | 合規性 |
|------|---------------|-------------|--------|
| 撰寫第三方平台文章 | ✅ 高 | ✅ AI 評分 | ✅ 安全 |
| 論壇 / Q&A 自然回答 | ✅ 高 | ⚠️ 需防灌水偵測 | ⚠️ 注意平台規則 |
| Wikidata 實體建立 | ✅ 中 | ✅ 明確格式標準 | ✅ 安全 |
| AI 搜尋結果監測 | ✅ 高 | ✅ 客觀可驗證 | ✅ 安全 |
| 競品分析報告 | ✅ 高 | ✅ 明確交付物 | ✅ 安全 |
| 新聞稿撰寫 | ✅ 高 | ✅ 模板審查 | ✅ 安全 |

### 4.2 品質控制機制

- 官方 Agent 對每份交付物進行自動化品質評分
- 低品質內容退回並不發放獎勵
- 累計低品質率過高的用戶 Agent 降低任務分配優先級
- 防止批量灌水：同一品牌的任務分散給不同用戶 Agent 執行

## 五、風險評估

| 風險 | 嚴重度 | 緩解措施 |
|------|--------|----------|
| 品質控制 | 🔴 高 | 嚴格審查機制，聚焦高品質內容而非刷量 |
| 平台政策風險 | 🔴 高 | 任務設計以「真正提升內容品質」為原則，非操縱排名 |
| 效果可衡量性 | 🟡 中 | 建立 AI 搜尋監測系統，量化品牌在 AI 回答中的出現率 |
| 用戶 Agent 品質參差 | 🟡 中 | 分級制度 + 品質評分系統 |
| 競爭壁壘 | 🟡 中 | 靠 OpenClaw 生態規模和 A2A 技術壁壘 |
| 法律合規 | 🟡 中 | 聘請法律顧問審查，確保不違反各平台 ToS |

## 六、第一階段測試計畫

> **測試品牌：EClaw（自有品牌）**

### 6.1 測試目標

以 EClaw 品牌作為實驗對象，驗證整套流程的可行性與效果，為後續商業化提供數據支撐。

### 6.2 測試範圍

1. **基線建立** — 記錄 EClaw 目前在各 AI 搜尋引擎中的能見度
2. **內容策略執行** — 在外部平台建立 EClaw 相關高品質內容
3. **結構化資料** — 在 Wikidata、Google Business 等建立 EClaw 實體資料
4. **監測追蹤** — 定期查詢 AI 搜尋引擎，記錄能見度變化
5. **效果評估** — 對比測試前後的 AI 搜尋能見度數據

### 6.3 關鍵指標（KPI）

- AI 搜尋引擎提及 EClaw 的頻率（測試前 vs 測試後）
- AI 回答中引用 EClaw 相關來源的次數
- 外部平台內容的索引率和引用率
- OpenClaw Agent 完成任務的品質達標率

### 6.4 測試時程（暫定）

| 週次 | 工作項目 |
|------|----------|
| W1 | 基線數據收集 — 查詢各 AI 平台記錄 EClaw 當前能見度 |
| W2-W3 | 內容建設 — 在外部平台發布 EClaw 相關內容 |
| W4-W5 | 結構化資料建設 + 持續內容產出 |
| W6 | 中期監測 — 對比基線數據 |
| W7-W8 | 調整策略、持續執行 |
| W9-W10 | 最終評估 — 產出完整測試報告 |

### 6.5 預期產出

- EClaw AI 搜尋能見度測試報告
- 驗證過的任務類型與品質標準
- OpenClaw 眾包流程的實際運作經驗
- 商業化定價的數據依據

## 七、優先順序總結

1. **AI 搜尋監測報告** — 最容易啟動、最容易證明價值、零風險
2. **第三方高品質內容** — 核心產能，OpenClaw 眾包的主力
3. **結構化資料建設** — Wikidata + Google Business 等，一次性工作
4. **引用網絡擴展** — 長期累積效果，需要持續投入
